# 颜色细分识别设计

## 目标

将当前图片识别中的粗粒度颜色标签升级为更符合人类搜索习惯的深浅颜色标签。识别结果应直接保存为实际颜色，例如 `浅绿色`、`深蓝色`、`红色`，而不是同时保存粗颜色和细颜色。

## 背景

当前系统在 `backend/app/services/color_analysis.py` 中通过图片主色 RGB 与固定颜色表计算距离，返回最接近的颜色标签。现有颜色表只有 `红色`、`绿色`、`蓝色` 等粗粒度标签，无法区分 `浅绿色`、`深绿色` 等用户实际会搜索的颜色。

## 范围

本阶段只改进基础颜色识别，不接入 YOLO、不做人脸/物体识别、不引入联网模型。

包含：

- 为主要色系增加浅色、标准色、深色三个等级。
- 黑色、白色作为独立颜色保留。
- 识别结果只返回一个最匹配的颜色标签。
- 搜索继续使用现有标签搜索能力，不新增搜索扩展规则。

不包含：

- 不加入 `肉色`、`肤色` 等泛化人体描述。
- 不让搜索 `绿色` 自动包含 `浅绿色` 和 `深绿色`。
- 不做多主色识别，例如一张图同时标记 `红色` 和 `蓝色`。

## 色系规则

主要色系覆盖：

- 浅红色 / 红色 / 深红色
- 浅橙色 / 橙色 / 深橙色
- 浅黄色 / 黄色 / 深黄色
- 浅绿色 / 绿色 / 深绿色
- 浅青色 / 青色 / 深青色
- 浅蓝色 / 蓝色 / 深蓝色
- 浅紫色 / 紫色 / 深紫色
- 浅粉色 / 粉色 / 深粉色
- 浅棕色 / 棕色 / 深棕色
- 浅灰色 / 灰色 / 深灰色
- 黑色
- 白色

黑色和白色不拆成深浅等级。

## 识别行为

系统仍然读取图片，缩略到小尺寸，统计出现最多的主色 RGB，并与扩展后的颜色表比较距离。

返回规则：

- 如果主色最接近浅绿色，返回 `浅绿色`。
- 如果主色最接近绿色，返回 `绿色`。
- 如果主色最接近深绿色，返回 `深绿色`。
- 不额外附加粗色标签。

示例：

| 图片主色 | 标签 |
| --- | --- |
| 浅绿色 | `浅绿色` |
| 普通绿色 | `绿色` |
| 深绿色 | `深绿色` |
| 浅蓝色 | `浅蓝色` |
| 深灰色 | `深灰色` |

## 搜索行为

因为颜色标签会保存到现有 annotation tags 中，当前图库搜索能力可以继续使用。

搜索示例：

- 搜 `浅绿色`：匹配标签为 `浅绿色` 的图片。
- 搜 `深绿色`：匹配标签为 `深绿色` 的图片。
- 搜 `绿色`：只匹配标签为 `绿色` 的图片，不自动匹配 `浅绿色` 或 `深绿色`。

如果未来需要“搜绿色包含浅绿色和深绿色”，应作为独立的同色系搜索扩展设计，不在本阶段实现。

## 测试要求

新增或更新测试覆盖：

- 浅绿色图片返回 `浅绿色`。
- 深绿色图片返回 `深绿色`。
- 普通绿色图片返回 `绿色`。
- 浅蓝色、深蓝色至少各有一个测试。
- 黑色和白色仍返回独立标签。
- 识别服务能把细分颜色标签持久化到数据库。
- 原有红色、黄色识别测试根据新色表保持通过或更新为明确色阶。

## 验收标准

- 后端颜色分析测试通过。
- 后端 API/识别相关测试通过。
- 识别一张浅绿色图片后，详情页或图库标签中出现 `浅绿色`。
- 搜索 `浅绿色` 能找到对应图片。
- 搜索 `绿色` 不要求找到 `浅绿色` 图片。
